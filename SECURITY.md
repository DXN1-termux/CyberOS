# Security Policy

## Supported Versions

We currently support and provide security updates for the following versions of CyberOS:

| Version | Supported          |
| ------- | ------------------ |
| 2.0.x   | :white_check_mark: |
| 1.x     | :x:                |

## Reporting a Vulnerability

We take the security of CyberOS seriously. If you believe you have found a security vulnerability, please report it to us by following these steps:

1. **Do not open a public issue.** We prefer to handle security reports privately to protect our users.
2. Email the maintainer at `security@cyberos.dev` (or open a private vulnerability report on GitHub if available).
3. Provide a detailed description of the vulnerability, including steps to reproduce and potential impact.

### Our Response Process

- We will acknowledge receipt of your report within 48 hours.
- We will investigate the issue and determine the appropriate fix.
- We will keep you informed of our progress.
- Once a fix is verified, we will release an update and credit you for the discovery (unless you prefer to remain anonymous).

## Scope

CyberOS is a tool for authorized security testing. The security of the *targets* you test with CyberOS is outside the scope of this project. This policy only applies to the CyberOS software itself.
